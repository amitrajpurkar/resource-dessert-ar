# Data Model: Flask Presentation Dashboard

**Feature**: 003-flask-presentation-dashboard
**Date**: 2026-03-28

---

## Entities

All entities are **read-only** — the Flask app consumes them from files generated by the analysis pipeline. No entity is written or mutated by the web app except as a side-effect of triggering notebook execution (which regenerates the source files).

---

### DesertScore

Represents one Jacksonville ZIP code's resource deprivation assessment.

**Source**: `reports/outputs/desert_scores.csv`

| Field | Type | Constraint | Notes |
|---|---|---|---|
| `zip_code` | `str` | 5-digit, unique | Primary identifier |
| `desert_rank` | `int` | ≥ 1, unique | 1 = most deprived |
| `desert_score` | `float` | [0.0, 100.0] | Higher = more deprived |
| `top_gap_category` | `str` | one of: `healthcare`, `food_access`, `parks`, `insurance` | Dominant supply gap |
| `total_population` | `int` | > 0 | 2020-2024 estimate |
| `poverty_rate` | `float` | [0.0, 1.0] | Fraction of population below poverty line |
| `demand_factor` | `float` | [0.0, 1.0] | Normalized demand weight |
| `supply_gap_healthcare` | `float` | [0.0, 1.0] | Normalized healthcare gap |
| `supply_gap_food` | `float` | [0.0, 1.0] | Normalized food access gap |
| `supply_gap_parks` | `float` | [0.0, 1.0] | Normalized parks gap |
| `supply_gap_insurance` | `float` | [0.0, 1.0] | Normalized insurance gap |

**Display subset** (top-10 table): `zip_code`, `desert_rank`, `desert_score`, `top_gap_category`, `total_population`, `poverty_rate`

### Intervention

Represents one simulated resource addition for a top-N underserved ZIP code.

**Source**: `reports/outputs/intervention_recommendations.json`

| Field | Type | Constraint | Notes |
|---|---|---|---|
| `zip_code` | `str` | 5-digit | Foreign key to DesertScore |
| `resource_type` | `str` | one of: `primary_care`, `food_access`, `parks`, `insurance_outreach` | Resource being simulated |
| `current_desert_score` | `float` | [0.0, 100.0] | Score before intervention |
| `simulated_desert_score` | `float` | [0.0, 100.0] | Score after intervention |
| `score_improvement` | `float` | ≥ 0.0 | Absolute point reduction |
| `pct_improvement` | `float` | [0.0, 100.0] | Percentage reduction |
| `population_impacted` | `int` | > 0 | Residents in the ZIP |
| `is_highest_impact` | `bool` | exactly one `true` across all rows | Flags the single best intervention |

**Total rows**: 20 (5 ZIPs × 4 resource types). Displayed sorted by `score_improvement` descending.

### ChartImage

A pre-rendered static PNG figure produced by the analysis pipeline.

**Source**: `reports/figures/`

| Field | Type | Notes |
|---|---|---|
| `filename` | `str` | Filename within `reports/figures/` |
| `section` | `str` | Which page section uses it: `analysis`, `observations`, `recommendations`, `category` |
| `caption` | `str` | Human-readable label shown beneath the image |
| `missing` | `bool` | True if the file does not exist on disk at render time |

**Known instances**:

| filename | section | caption |
|---|---|---|
| `desert_scores_bar_chart.png` | `analysis` | Top 10 Most Underserved Jacksonville ZIP Codes |
| `preventative_asset_vs_health_outcome.png` | `observations` | Preventative Asset vs Health Outcome (regression) |
| `intervention_impact_heatmap.png` | `recommendations` | Gap-Closure Simulation: Projected Improvement by Intervention |
| `category_healthcare_view.png` | `category` | Healthcare Supply Gap by ZIP Code |
| `category_food_access_view.png` | `category` | Food Access Supply Gap by ZIP Code |
| `category_parks_view.png` | `category` | Parks Supply Gap by ZIP Code |
| `category_insurance_view.png` | `category` | Insurance Coverage Gap by ZIP Code |

### NotebookRunStatus

Transient in-memory state tracking the background notebook execution. Not persisted to disk.

| Field | Type | Values | Notes |
|---|---|---|---|
| `status` | `str` | `idle` \| `running` \| `complete` \| `error` | Current run state |
| `message` | `str` | Empty string or last 500 chars of stderr | Error detail on failure |

**Lifecycle**:
```
idle → running  (POST /api/regenerate, when status != "running")
running → complete  (subprocess returncode == 0)
running → error     (subprocess returncode != 0)
complete → running  (subsequent POST /api/regenerate)
error    → running  (subsequent POST /api/regenerate — allows retry)
```

**Thread safety**: Protected by `threading.Lock`. The `status` and `message` fields are only written inside the lock.

## File Dependency Map

```
app startup
    └── reads desert_scores.csv     → DesertScore[] (top-10 for initial render)
    └── reads intervention_recommendations.json  → Intervention[] (all 20, best highlighted)
    └── checks existence of 7 output files       → ChartImage.missing flags

GET /figures/<filename>
    └── serves from reports/figures/<filename>

GET /map
    └── serves reports/outputs/resource_desert_map.html

POST /api/regenerate
    └── spawns thread → runs nbconvert → regenerates all 7 output files
    └── updates NotebookRunStatus in memory

GET /api/regenerate/status
    └── reads NotebookRunStatus (lock-protected)
```
